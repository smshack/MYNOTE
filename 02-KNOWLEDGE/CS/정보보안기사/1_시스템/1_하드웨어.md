## 1. CPU
- 입력장치로 부터 자료를 받아 연산하고 그 결과를 출력 장치로 보내는 일련의 과정을 제어 및 조장하는 핵심 장치

#### CPU 구성요소

| ALU (산술논리연산장치) | 산술 연산(+,-,*,/) 및 논리 연산(AND, OR 등) 수행 | 계산     |
| -------------- | ------------------------------------ | ------ |
| 레지스터           | CPU 내부의 초고속 임시 저장소                   | 임시 저장  |
| 제어장치 (CU)      | 명령어 해석 및 각 장치 제어                     | 제어, 해석 |
| 내부 버스          | CPU 내부에서 데이터 이동 통로                   | 데이터 전달 |
제어장치가 명령 해석 → 레지스터/ALU로 처리 → 내부버스로 이동

---

#### 내부 레지스터 종류

| 레지스터                          | 설명                          | 역할 요약  |
| ----------------------------- | --------------------------- | ------ |
| PC (Program Counter)          | 다음 실행할 명령어 주소 저장            | 다음 주소  |
| IR (Instruction Register)     | 현재 실행 중인 명령어 저장             | 현재 명령  |
| MAR (Memory Address Register) | 접근할 메모리 주소 저장               | 주소 지정  |
| MDR (Memory Data Register)    | 메모리에서 읽거나 쓸 데이터 저장          | 데이터 저장 |

- **MAR = 주소 / MDR = 데이터 (무조건 구분!)**
    
- **PC = 다음 / IR = 현재**
    

---

#### 실행 파일 분석(리버싱)에서 중요한 레지스터

> (x86 / x64 기준 – 보안, 리버싱, 디버깅에서 자주 나옴)

### 📌 일반 목적 레지스터 (General Purpose)

| 레지스터      | 설명                  | 중요도 |
| --------- | ------------------- | --- |
| EAX / RAX | 연산 결과 저장 (가장 많이 사용) | ⭐⭐⭐ |
| EBX / RBX | 기본 데이터 저장           | ⭐⭐  |
| ECX / RCX | 반복문, 카운터            | ⭐⭐⭐ |
| EDX / RDX | 연산 보조               | ⭐⭐  |

✔ 레지스터 핵심 4개  
👉 **PC / IR / MAR / MDR**

✔ 리버싱 핵심

#### ✅ 1. 버스 시스템 정의

| 항목      | 설명                                   |
| ------- | ------------------------------------ |
| 버스(Bus) | 컴퓨터 내부에서 **데이터, 주소, 제어 신호를 전달하는 통로** |
| 역할      | CPU ↔ 메모리 ↔ 입출력장치 간 연결               |
| 특징      | 여러 장치가 **공유**하는 통신 경로                |

➡️ **“버스 = 컴퓨터 내부 도로 (데이터 이동 통로)”**

---

#### ✅ 2. 버스 종류

| 버스 종류               | 역할                 | 방향              | 핵심 포인트    |
| ------------------- | ------------------ | --------------- | --------- |
| 데이터 버스 (Data Bus)   | 실제 데이터 전달          | 양방향             | 읽기/쓰기 둘 다 |
| 주소 버스 (Address Bus) | 메모리 주소 전달          | 단방향 (CPU → 메모리) | 주소 지정     |
| 제어 버스 (Control Bus) | 제어 신호 전달 (읽기/쓰기 등) | 양방향             | 동작 제어     |

---

### 🎯 시험 핵심 구분

| 구분     | 키워드      |
| ------ | -------- |
| 데이터 버스 | 값 이동     |
| 주소 버스  | 위치 지정    |
| 제어 버스  | 읽기/쓰기 제어 |

👉 암기 팁  
➡️ **“데값 / 주위 / 제컨”**  
(데이터=값, 주소=위치, 제어=컨트롤)

---

#### ✅ 3. CPU 명령어 실행 주기 (Instruction Cycle)

> CPU가 프로그램을 실행하는 반복 과정

---

### 📌 전체 흐름

```
인출(Fetch) → 해독(Decode) → 실행(Execute) → 저장(Write-back)
```

---

## ✅ 4. 단계별 상세 설명

|단계|영어|설명|사용 레지스터|
|---|---|---|---|
|인출|Fetch|메모리에서 명령어 가져옴|PC, MAR, MDR|
|해독|Decode|명령어 해석|IR|
|실행|Execute|연산 수행|ALU, 레지스터|
|저장|Write-back|결과 저장|레지스터 / 메모리|

---

## ✅ 5. 실행 과정 흐름 (시험 중요 ⭐)

|순서|동작|
|---|---|
|1|PC → 다음 명령어 주소|
|2|MAR ← 주소 전달|
|3|MDR ← 메모리 데이터 읽기|
|4|IR ← 명령어 저장|
|5|제어장치 → 명령어 해석|
|6|ALU → 연산 수행|
|7|결과 저장|

---

## 🎯 초압축 암기

```
버스 = 데이터 / 주소 / 제어

데이터 → 값
주소 → 위치
제어 → 읽기/쓰기

명령어 실행:
Fetch → Decode → Execute → Write-back

PC → MAR → MDR → IR → ALU
```

---

## 🔥 시험에 잘 나오는 포인트

✔ 주소버스는 **단방향 (CPU → 메모리)**  
✔ 데이터버스는 **양방향**  
✔ Fetch 단계에서 **PC 사용**  
✔ Decode 단계에서 **IR 사용**  
✔ Execute 단계에서 **ALU 사용**

# 📘 메모리 & 캐시 & 페이지 교체 정리 (시험 대비)

---

## ✅ 1. 메모리 (Memory)

| 항목  | 설명                          |
| --- | --------------------------- |
| 메모리 | 데이터를 저장하는 장치                |
| 종류  | 주기억장치(RAM), 보조기억장치(HDD/SSD) |
| 특징  | 속도 ↔ 용량 ↔ 가격 트레이드오프 존재      |

👉 한 줄  
➡️ **“빠르면 비싸고 작고, 느리면 싸고 크다”**

---

## ✅ 2. 기억장치 계층 구조

```
레지스터 → 캐시 → 메인메모리 → 보조기억장치
(빠름/비쌈 ↑)           (느림/쌈 ↓)
```

---

### 📌 표 정리

|계층|속도|용량|비용|위치|
|---|---|---|---|---|
|레지스터|가장 빠름|매우 작음|매우 비쌈|CPU 내부|
|캐시|빠름|작음|비쌈|CPU 근처|
|메인메모리|보통|큼|중간|RAM|
|보조기억장치|느림|매우 큼|저렴|HDD/SSD|

---

## ✅ 3. 메모리 계층 구조 이유

|이유|설명|
|---|---|
|속도 향상|자주 쓰는 데이터는 빠른 메모리에 저장|
|비용 절감|전체를 고속 메모리로 구성 불가능|
|효율성|접근 시간 최소화|

👉 핵심  
➡️ **“자주 쓰는 건 가까이 둔다 (지역성)”**

---

## ✅ 4. 캐시 메모리 (Cache Memory)

|항목|설명|
|---|---|
|정의|CPU와 메모리 사이의 고속 버퍼|
|목적|메모리 접근 속도 향상|
|특징|지역성(Locality) 활용|

---

### 📌 지역성 종류

|종류|설명|
|---|---|
|시간 지역성|최근 사용 데이터 다시 사용|
|공간 지역성|근처 데이터 사용|

---

## ✅ 5. 캐시 메모리 사상 방식 (Mapping)

|방식|설명|특징|
|---|---|---|
|직접 사상 (Direct)|특정 위치에만 저장|빠름, 충돌 많음|
|연관 사상 (Associative)|아무 위치나 저장|느림, 충돌 적음|
|집합 연관 (Set-Associative)|그룹 내 자유 저장|절충|

👉 암기  
➡️ **직 = 빠/충많 / 연 = 느/충적 / 집 = 중간**

---

## ✅ 6. 캐시 메모리 관리 방식 (쓰기 정책)

|방식|설명|특징|
|---|---|---|
|Write-through|캐시 + 메모리 동시에 기록|일관성 좋음, 느림|
|Write-back|캐시에만 기록 후 나중 반영|빠름, 복잡|

---

## ✅ 7. 캐시 교체 알고리즘

|알고리즘|설명|
|---|---|
|FIFO|먼저 들어온 것 제거|
|LRU|가장 오래 안 쓴 것 제거 ⭐|
|LFU|사용 횟수 적은 것 제거|
|Random|랜덤 제거|

👉 핵심  
➡️ **LRU = 시험 단골**

---

## ✅ 8. 페이지 교체 알고리즘 문제점

|문제|설명|
|---|---|
|페이지 부재 (Page Fault)|필요한 페이지 없음|
|스래싱 (Thrashing)|페이지 교체 과다로 성능 급락 ⭐|
|오버헤드 증가|교체 비용 발생|

---

## ✅ 9. 페이지 교체 문제 해결 방안

|해결 방법|설명|
|---|---|
|워킹셋 (Working Set)|자주 사용하는 페이지 유지|
|페이지 프레임 증가|메모리 확장|
|LRU 사용|효율적 교체|
|지역성 활용|접근 패턴 최적화|

---

## 🎯 초압축 암기 (시험 직전용)

```
계층구조:
레지스터 → 캐시 → 메모리 → 디스크

이유:
속도 + 비용 → 지역성 활용

캐시:
시간지역성 / 공간지역성

사상:
직접 / 연관 / 집합연관

교체:
FIFO / LRU / LFU

문제:
Page Fault / Thrashing

해결:
Working Set / LRU / 메모리 증가
```

---

## 🔥 시험 핵심 TOP

✔ LRU 반드시 기억  
✔ Thrashing = 성능 급락  
✔ 지역성 2가지 (시간/공간)  
✔ Direct vs Associative 차이  
✔ Write-through vs Write-back

|고정 분할|미리 나눔|

## ✅ 가상 메모리 관리 기법

### 📌 1) 페이징
- 논리주소의 고정된 페이지 라고 불리는 블록들로 분할 관리
- 각각의 페이지는 물리 메모리의 프레임과 맵핑
- 페이지를 가리키는 논리 주소에서 프레임을 가리키는 물리조소로 변환

|항목|설명|
|---|---|
|특징|고정 크기 페이지|
|장점|외부 단편화 없음|
|단점|내부 단편화 발생|

---

### 📌 2) 세그멘테이션
- 메모리를 세그먼트 세트로 나눠 관리
- 시그먼트: 시작주소(베이스 어드레스)와 세그먼트의 크기를 지정하는 길이 값으로 구성
- 가상 메모리 주소는 세그먼트 번호와 변위 값으로 구성
- 세그먼트 테이블에서 시작 주소를 획득하고 변위값(offset) 과 결합하여 물리 메모리 주소 산출

|항목|설명|
|---|---|
|특징|논리 단위 (코드, 데이터 등)|
|장점|사용자 관점|
|단점|외부 단편화|

---

### 📌 3) 페이지드 세그멘테이션

|항목|설명|
|---|---|
|특징|세그먼트 + 페이지|
|장점|단점 보완|
|단점|구조 복잡|

## 🎯 가상 메모리 초압축

```id="vm01"

정책:
얼마(할당) / 언제(호출) / 어디(배치) / 뭐(교체)
```

- 내부: 메모리 할당시 프로세스가 필요한 양보다 더 큰 메모리 할당 프로세스 사용 메모리 공간이 낭비
- 외부: 메모리 할당 해제 작업 반복 중 할당된 메모리와 메모리 사이에 중간 중간 사용하지 않는 작은 메모리가 생겨 총 메모리 공간은 충분하지만 실제 할당 가능 공간 부족

## ✅ 입출력 방법 (CPU 경유 여부)

|---|---|---|
|CPU 상태|계속 확인 (Polling)|필요 시만 처리|
|효율|낮음|높음|
|특징|단순|CPU 절약|

👉 암기  
➡️ **Polling vs Interrupt**

## ✅ DMA (Direct Memory Access)

### 📌 정의

|항목|설명|
|---|---|
|DMA|CPU 없이 메모리 ↔ I/O 직접 전송|

---

### 📌 동작 방식

|단계|설명|
|---|---|
|1|CPU가 DMA에 작업 지시|
|2|DMA가 직접 데이터 전송|
|3|완료 후 인터럽트 발생|

👉 핵심  
➡️ **“CPU 개입 없음 → 속도 향상”**

---

### 📌 구성요소

|구성|설명|

## 🎯 I/O 초압축 암기

```id="io01"
입출력:
Programmed / Interrupt / DMA

Programmed = CPU 계속 사용
Interrupt = 필요할 때만
DMA = CPU 없이 직접 전송

DMA:
CPU 지시 → DMA 처리 → 인터럽트

I/O Processor:
입출력 전용 CPU
```
