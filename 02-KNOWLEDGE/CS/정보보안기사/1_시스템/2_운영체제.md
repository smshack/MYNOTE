### 📌 2. 프로세스 관리

---

#### ✅ 프로세스 vs 스레드

| 구분    | 프로세스                                 | 스레드                                |
| ----- | ------------------------------------ | ---------------------------------- |
| 정의    | 레지스터, 스택, 포인터, 실행중인 프로그램, 데이터 등      | 제어의 흐름을 의미하는 것으로 프로세스에서 실행의 개념만 분리 |
| 상호 통신 | system call<br>call 종료 시 까지 전체 자원 블록 | library call<br>요청 스레드만 블록         |
| 처리 방식 | 자원 할당을 위한 기본 구분 단위                   | CPU를 이용하는 기본 작업<br>단위로 구분          |
| 메모리   | 독립                                   | 공유                                 |
| 비용    | 큼                                    | 작음                                 |

---

#### ✅ 프로세스 상태 전이
> 운영체제- 프로세스 실행 제어, 프로세스에 대한 정보 유지

> 프로세스 실행 결정 및 필요한 시스템 자원을 할당하는 과정

> 작업 스케줄러, 프로세스 스케줄러

```id="pst01"
생성 → 준비 → 실행 → 대기 → 종료
```

---

#### ✅ 문맥교환
- **PCB**: 운영체제가 프로세스를 제어하기 위해 정보를 저장해 놓는 곳으로, 프로세스의 상태 정보를 저장하는 구조체
> CPU 가 이전의 프로세스 상태를 PCB에 보관하고, 또는 다른 프로세스의 정보를 PCB에서 읽어 레지스터에 적재하는 과정


|항목|설명|
|---|---|
|정의|CPU가 다른 프로세스로 전환|
|특징|오버헤드 발생|

### 📌 3. CPU 스케줄링

---

#### ✅ 정의 / 목적

| 구분  | 내용                                                                               |
| --- | -------------------------------------------------------------------------------- |
| 정의  | 컴퓨터의 자원을 효율적으로 사용하기 위한 정책을 계획<br>특정 자원을 요청하고 있는 프로세스를 대상으로 cpu 자원을 할당해 주는 순서를 정함 |
| 목적  | cpu 활용 극대화/ 응답시간 단축/ 공평한 자원 활용/ 멀티테스킹 효율화                                        |

---

#### ✅ 스케줄러 종류

| 종류  | 역할                                                                 |
| --- | ------------------------------------------------------------------ |
| 장기  | 상위 스케줄링, 작업 스케줄링(Job 스케줄링)<br>어떤 작업이 시스템의 자원들을 차지할 것인지 결정(큐에 적재)   |
| 중기  | 어떤 프로세스들이 cpu를 할당 받을 것인지 결정<br>cpu를 사용하려는 프로세스 간 중재하여 일시 보류 & 재활성화 |
| 단기  | 하위 스케줄링, cpu 스케줄링, 프로세스 스케줄링이라고 함<br>cpu 스케줄러인 dispatcher에 의해 동작   |

---

#### ✅ 점유 방식

| 구분      | 선점                                   | 비선점                                                     |
| ------- | ------------------------------------ | ------------------------------------------------------- |
| 개념      | 프로세스가 cpu 점유 중에서도 다른 프로세스가 cpu 점유 가능 | 프로세스가 cpu를 해제할 때까지 다른 프로세스는 대기                          |
| 장점      | 빠른 응답, 모바일, RTOS<br>대화식 시분할 적합       | 응답 시간 예상 용이<br>Batch Process 적합<br>프로세스에 대한 요구를 공정하게 처리 |
| 단점      | 오버헤드 발생                              | 짧은 작업에도 장기간 대기하는 경우가 발생                                 |
| 스케줄링 기법 | 라운드로빈, SRT                           | FCFS, SJF,HRN                                           |

---

#### ✅ CPU 스케줄링 기법

| 기법                             | 특징                                                                                                                                             | 점유 방식 |
| ------------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------- | ----- |
| FCFS(First Come First Service) | 먼저 온 순서<br>프로세스가 cpu를 차지하면 완료 될때 까지 수행                                                                                                         | 비선점   |
| SJF(Shortes Job First)         | 짧은 작업                                                                                                                                          | 비선점   |
| 라운드 로빈(Round Robin)            | FCFS에 의해 프로세스들이 내보내지며 각 프로세스는 같은 크기의 CPU 시간을 할당<br>CPU 시간이 만료될 때까지 처리를 완료하지 못하면 CPU는 대기중인 다음 프로세스로 넘어가며, 실행 중이던 프로세스는 준비 완료 리스트의 가장 뒤로 보내짐     | 선점형   |
| SRT(Shortest Remaining Time)   | SJF 기법 사용 실행중인 프로세스라도 남은 처리 시간이 더 짧다고 판단되는 프로세스가 생기면 언제라도 실행중인 프로세스가 선점<br>SJF +  선점                                                           | 선점형   |
| Multi Level Queue              | 여러 종류의 그룹으로 나누어 여러 개의 큐를 이용하는 스케줄링<br>그룹화된 작업들은 각각의 준비 큐에 넣어두고 독자적인 스케줄링 알고리즘에 따라 CPU를 할당                                                      |       |
| Multi Level Feedback Queue     | 입출력 위주와 cpu 위주인 프로세스의 특성에 따라 서로 다른 타임 슬라이스를 부여<br>새로운 프로그램이 들어오면 높은 우선순위를 할당해 주어 단계 1에서 즉시 수행, 점차 낮은 우선 순의를 부여하며 나중에는 그 작업이 완료될 때까지 라운드로빈으로 순환 | 하이브리드 |

### 📌 5. 교착상태


#### ✅ 개념

| 항목  | 설명                                                          |
| --- | ----------------------------------------------------------- |
| 정의  | 둘 이상의 프로세스가 더이상 계속 할 수 없는 어떤 특정한 사건(자원의 할당과 해제)을 기다리고 있는 상태 |

---

#### ✅ 발생 조건 (⭐)

| 조건   | 설명                                                                      |
| ---- | ----------------------------------------------------------------------- |
| 상호배제 | 프로세스들이 자원을 배타적으로 점유<br>다른 프로세서들이 자원 사용 불가<br>한번에 한 프로세스만 자원 사용 가능       |
| 점유대기 | 부분 할당, 다른 종류의 자원을 부가적으로 요구하면서, 이미 어떤 자원을 점유하고 있음                        |
| 비선점  | 자원들은 점유하고 있는 프로세스는 중간에 해제 되지 않음                                         |
| 환형대기 | 프로세스와 자원들이 원형을 이루며, 각 프로세스는 자신에게 할당된 자원을 가지면서, 상대방 프로세스의 자원을 상호 요청하는 경우 |

👉 암기  
➡️ **상점비순**

---

#### ✅ 대응 방법

| 방법  | 설명                                                        |                                           |
| --- | --------------------------------------------------------- | ----------------------------------------- |
| 예방  | 점유와 대기조건의 부정<br>비선점 조건의 부정<br>환형대기 조건의 부정<br>상호 배제 조건의 부정 | 교착 상태의 필요조건을 부정함으로써 발생하지 않도록 미리 예방        |
| 회피  | 은행원 알고리즘                                                  | 가능성을 배제하지 않고, 적절하게 피해나가는 방법               |
| 발견  | 알고리즘<br>자원 할당 그래프                                         | 발생을 허용하고, 발생 시 원인을 규명하여 해결하는 방법           |
| 회복  | 프로세스 중지<br>자원 선점                                          | 교착상태 발견 후 환형대기를 배제시키거나 자원을 중단하는 메모리 할당 기법 |

### 📌 6. 디스크 관리

---

#### ✅ 디스크 접근 시간
- 가장 많이 사용되는 보조기억장치 
- 주기억장치보다 많은 양의 데이터를 저장 가능

| 요소              | 설명                                   |
| --------------- | ------------------------------------ |
| 탐색 시간 (Seek)    | 현 위치에서 특정 실린더로 디스크 헤드가 이동하는데 소요되는 시간 |
| 회전 지연 (Latency) | 가고자 하는 섹터가 디스크 헤드까지 도달하는데 걸리는 시간     |
| 전송 시간           | 데이터를 전송하는데 걸리는 시간                    |

👉 공식 느낌  
➡️ **접근시간 = 탐색 + 회전 + 전송**

---

#### ✅ 디스크 스케줄링

| 알고리즘                           | 특징                                                                                         |
| ------------------------------ | ------------------------------------------------------------------------------------------ |
| FCFS(First Come First Service) | 가장 먼저 도착한 요청 우선적으로 처리                                                                      |
| SSTF(Shortest-Seek Time First) | 탐색 거리가 가장 짧은 트랙에 대한 요청을 먼저 서비스함                                                            |
| SCAN(엘리베이터 알고리즘)               | Head가 이동하는 방향의 모든 요청을 서비스하고, 끝까지 이동한 후 역방향의 요청을 서비스함                                       |
| C-SCAN(Circular-SCAN)          | 바깥에서 안쪽으로 이동 끝에 도달하면 바깥쪽으로 이동하여 요청을 다시 처리                                                  |
| C-LOOK(Circular-Look)          | C-SCAN 보완, 대기시간을 좀더 균형 있게<br>헤드 이동 방향의 마지막 입출력 요청을 처리한 후 디스크 헤드를 처음 위치로 이동하여 다음 입출력 요청을 처리 |
|                                |                                                                                            |

### 📌 8. RAID

---

#### ✅ 개념

| 항목   | 설명                                                                                                                                                               |
| ---- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| RAID | 디스크 고장 시 그대로 복구할 수 있도록 2개 이상의 디스크에 데이터를 저장하는 기술<br>디스크의 기계적인 장애로 부터 사용자의 데이터를 안정적으로 지킬 수 있음<br>저용량, 저성능, 저가용성인 디스크를 배열 구조로 중복 구성함으로써 교용량, 고성능, 고가용성 디스크를 대체하고자 함 |

---

| 종류     | 특징           |
| ------ | ------------ |
| RAID 0 | 속도 ↑ (스트라이핑) |
| RAID 1 | 미러링 (안정성) ⭐  |
| RAID 5 | 패리티 사용 ⭐     |
| RAID 6 | 이중 패리티       |
