
# 📘 `/etc/passwd` 파일 구조 정리

---

#### ✅ 1. 개념

|항목|설명|
|---|---|
|`/etc/passwd`|사용자 계정 정보를 저장하는 파일|
|특징|모든 사용자가 읽기 가능 (보안상 비밀번호는 없음)|

👉 핵심  
➡️ **“사용자 정보 저장, 비밀번호는 없음 (/etc/shadow에 있음)”**

---

#### ✅ 2. 파일 구조 형식

```bash
username:password:UID:GID:comment:home:shell
```

👉 구분자  
➡️ **콜론(:)으로 구분**

---

#### ✅ 3. 필드별 상세 설명

| 순서  | 필드       | 설명            |
| --- | -------- | ------------- |
| 1   | username | 사용자 계정 이름     |
| 2   | password | 암호 자리 (보통 x)  |
| 3   | UID      | 사용자 ID        |
| 4   | GID      | 그룹 ID         |
| 5   | comment  | 사용자 설명 (이름 등) |
| 6   | home     | 홈 디렉터리        |
| 7   | shell    | 로그인 셸         |

---

#### ✅ 4. 실제 예시

```bash
root:x:0:0:root:/root:/bin/bash
```

---

### 📌 해석

| 필드       | 값         | 의미           |
| -------- | --------- | ------------ |
| username | root      | 관리자 계정       |
| password | x         | shadow 파일 참조 |
| UID      | 0         | 관리자          |
| GID      | 0         | root 그룹      |
| comment  | root      | 설명           |
| home     | /root     | 홈 디렉터리       |
| shell    | /bin/bash | 사용 셸         |

---

#### ✅ 5. UID 의미 (시험 자주 나옴 ⭐)

| UID     | 의미         |
| ------- | ---------- |
| 0       | root (관리자) |
| 1~999   | 시스템 계정     |
| 1000 이상 | 일반 사용자     |

---

#### ✅ 6. password 필드 의미

| 값      | 의미                   |
| ------ | -------------------- |
| x      | `/etc/shadow`에 암호 저장 |
| * 또는 ! | 로그인 불가               |

---

#### ✅ 7. /etc/shadow와 차이

| 구분  | passwd   | shadow   |
| --- | -------- | -------- |
| 내용  | 사용자 정보   | 암호       |
| 권한  | 모두 읽기 가능 | root만 접근 |
| 보안  | 낮음       | 높음       |

---

## 🎯 초압축 암기

```id="passwd03"
구조:
계정:비번:UID:GID:설명:홈:셸

비번:
x → shadow

UID:
0=root
1000↑=일반

핵심:
passwd=정보 / shadow=암호
```

---

## 🔥 시험 핵심 포인트

✔ 필드 순서 (7개) ⭐  
✔ password = x 의미 ⭐  
✔ UID 0 = root ⭐  
✔ shadow 파일과 차이 ⭐
시험 대비용으로 **README + 표 + 계산 문제 + 흐름도**까지 한 번에 정리해줄게.

# 📘 1. `/etc/shadow` 파일 구조

---

#### ✅ 1. 개념

| 항목            | 설명                       |
| ------------- | ------------------------ |
| `/etc/shadow` | 사용자 **암호 및 보안 정보 저장 파일** |
| 특징            | root만 접근 가능 (보안 핵심 파일)   |

👉 한 줄  
➡️ **“암호는 shadow에 저장”**

---

#### ✅ 2. 파일 구조 형식

```bash
username:password:lastchg:min:max:warn:inactive:expire:reserved
```

---

#### ✅ 3. 필드 설명

| 순서  | 필드       | 설명       |
| --- | -------- | -------- |
| 1   | username | 사용자 계정   |
| 2   | password | 암호 (암호화) |
| 3   | lastchg  | 마지막 변경일  |
| 4   | min      | 최소 변경일   |
| 5   | max      | 최대 사용일   |
| 6   | warn     | 만료 전 경고  |
| 7   | inactive | 비활성 기간   |
| 8   | expire   | 계정 만료일   |
| 9   | reserved | 예약       |

---

#### ✅ 4. password 필드 의미

| 값   | 의미         |
| --- | ---------- |
| $6$ | SHA-512 해시 |
| !   | 로그인 불가     |
| *   | 계정 비활성     |

---

#### ✅ 5. 예시

```bash
root:$6$abc123...:19000:0:99999:7:::
```

👉 해석

- SHA-512 암호
    
- 19000일에 변경
    
- 7일 전 경고
    

---

## 🎯 초압축 암기

```id="shadow03"
shadow:
계정:암호:변경:최소:최대:경고:비활성:만료

암호:
$6$ = SHA512
! = 로그인 불가
```

# 📘 2. chmod / umask 계산 문제

---

#### ✅ 1. chmod 기본

| 권한  | 값   |
| --- | --- |
| r   | 4   |
| w   | 2   |
| x   | 1   |

---

#### ✅ 예제

```bash
chmod 755 file
```

| 대상  | 권한      |
| --- | ------- |
| 사용자 | 7 (rwx) |
| 그룹  | 5 (r-x) |
| 기타  | 5 (r-x) |

---

#### ✅ 2. umask 계산

👉 공식

```id="umask01"
기본값 - umask = 최종 권한
```

| 기본값       |
| --------- |
| 파일: 666   |
| 디렉터리: 777 |

---

#### ✅ 예제 문제

### 📌 문제 1

👉 umask = 022 일 때

| 항목   | 계산              |
| ---- | --------------- |
| 파일   | 666 - 022 = 644 |
| 디렉터리 | 777 - 022 = 755 |

---

### 📌 문제 2

👉 umask = 027

| 항목   | 결과  |
| ---- | --- |
| 파일   | 640 |
| 디렉터리 | 750 |

---

## 🎯 초압축 암기

```id="chmod02"
r=4 w=2 x=1

파일=666
디렉터리=777

umask:
빼기 계산

022 → 644 / 755
```

# 📘 3. 리눅스 계정 생성 흐름

---

#### ✅ 전체 흐름

```id="user01"
useradd → passwd → /etc/passwd → /etc/shadow → 홈디렉터리 생성
```

---

#### ✅ 단계별 설명

| 단계        | 설명        |
| --------- | --------- |
| useradd   | 계정 생성     |
| passwd    | 비밀번호 설정   |
| passwd 파일 | 사용자 정보 등록 |
| shadow 파일 | 암호 저장     |
| 홈 디렉터리    | 자동 생성     |

---

#### ✅ 관련 파일 변화

| 파일          | 내용        |
| ----------- | --------- |
| /etc/passwd | 사용자 정보 추가 |
| /etc/shadow | 암호 정보 추가  |
| /etc/group  | 그룹 정보     |

---

#### ✅ 주요 명령어

```bash
useradd testuser
passwd testuser
id testuser
```

---

## 🎯 초압축 암기

```id="user03"
계정 생성:

shadow = 암호

파일:

```
