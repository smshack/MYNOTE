
## 📘 cron & 리눅스 악성코드 탐지 정리

### 📌 1. cron


#### ✅ 정의

|항목|설명|
|---|---|
|cron|**정기적으로 작업을 자동 실행하는 스케줄러**|

👉 한 줄  
➡️ **“예약 작업 실행 도구”**

---

#### ✅ 파일 구조 ⭐

```bash
* * * * * command
```

##### 📌 필드 의미

| 순서  | 필드  | 범위   |
| --- | --- | ---- |
| 1   | 분   | 0~59 |
| 2   | 시   | 0~23 |
| 3   | 일   | 1~31 |
| 4   | 월   | 1~12 |
| 5   | 요일  | 0~7  |

---

##### 📌 예제

```bash
0 2 * * * backup.sh
```

👉 의미  
➡️ 매일 2시에 실행

---

#### ✅ cron 파일 위치

| 파일           | 설명       |
| ------------ | -------- |
| /etc/crontab | 시스템 전체   |
| /etc/cron.d/ | 개별 설정    |
| crontab -e   | 사용자 설정 ⭐ |

#### ✅ 옵션

```bash
crontab -l
crontab -e
crontab -r
```

|옵션|설명|
|---|---|
|-l|목록|
|-e|수정|
|-r|삭제|

---

#### ✅ 설정 방법

```bash
crontab -e
```

👉 추가 후 저장

### 📌 3. 리다이렉션 (Redirection)

#### ✅ 개념

|기호|의미|
|---|---|
|>|덮어쓰기|
|>>|추가|
|<|입력|

##### 📌 예제

```bash
ls > file.txt
```

---

### 📌 4. 파이프 (Pipe)

#### ✅ 개념

| 기호  | 의미                                 |
| --- | ---------------------------------- |
| \|  | 어떤 명령어의 실행 결과를 다른 명령어의 입력으로 전달하는 것 |

##### 📌 예제

```bash
ps -ef | grep ssh
```

👉 의미  
➡️ 프로세스 필터링

---

### 📌 5. find 명령어 ⭐


#### ✅ 기본 구조

```bash
find [경로] [옵션] [조건]
```

#### ✅ 주요 옵션

|옵션|설명|
|---|---|
|-name|이름|
|-type|파일 종류|
|-perm|권한|
|-size|크기|

#### ✅ 예제

```bash
find / -name test.txt
find / -perm -4000
find / -type f
```

---
##### 📌 중요 패턴

|명령어|의미|
|---|---|
|find / -perm -4000|SUID 탐지 ⭐|
|find / -name "*.sh"|스크립트 찾기|
|find / -size +100M|큰 파일|

## 🎯 악성코드 탐지 핵심

```id="mal01"
find / -perm -4000
find / -name "*.sh"
ps -ef | grep
```
