# 📘 Windows NTFS 파일 시스템 정리

---
#### 📌 1. NTFS 개요

| 항목   | 설명                |
| ---- | ----------------- |
| NTFS | Windows 기본 파일 시스템 |
| 특징   | 보안, 안정성, 대용량 지원   |

👉 한 줄  
➡️ **“보안 + 안정성 강화 파일 시스템”**

---

#### 📌 2. 파일 시스템 주요 특징 ⭐

| 특징     | 설명             |
| ------ | -------------- |
| 권한 관리  | ACL 기반 접근 제어 ⭐ |
| 저널링    | 장애 시 복구 가능     |
| 대용량 지원 | 큰 파일/디스크 지원    |
| 압축     | 파일 압축 기능       |
| 암호화    | EFS 지원 ⭐       |

➡️ **ACL / 저널링 / 암호화**

---

#### 📌 3. EFS (Encrypting File System)

| 항목  | 설명           |
| --- | ------------ |
| EFS | 파일 단위 암호화 기능 |

| 특징         | 설명           |
| ---------- | ------------ |
| 사용자 기반     | 사용자 계정으로 암호화 |
| 자동 암복호화    | 접근 시 자동 처리   |
| 파일 단위      | 특정 파일만 가능    |
| NTFS에서만 사용 | 다른 FS 불가     |

---

👉 한 줄  
➡️ **“파일 단위 암호화 (사용자 기반)”**

---

#### 📌 4. BitLocker vs EFS ⭐ (시험 단골)

| 항목     | BitLocker | EFS    |
| ------ | --------- | ------ |
| 암호화 대상 | 전체 디스크 ⭐  | 파일/폴더  |
| 기반     | 볼륨        | 파일     |
| 사용자    | 관계 없음     | 사용자 기반 |
| 보안 수준  | 매우 높음     | 중간     |
| 적용     | OS 보호     | 데이터 보호 |
➡️ **BitLocker = 전체 / EFS = 파일**

---

#### 📌 5. NTFS 파일 구조 ⭐

```id="ntfs01"
Boot Sector → MFT → Data Area
```

| 구성                      | 설명         |
| ----------------------- | ---------- |
| Boot Sector             | 파일 시스템 정보  |
| MFT (Master File Table) | 파일 정보 저장 ⭐ |
| Data Area               | 실제 데이터     |

---

## ✅ MFT 특징 ⭐

| 항목       | 설명          |
| -------- | ----------- |
| 파일 정보 저장 | 모든 파일 정보 관리 |
| 메타데이터    | 파일 속성 포함    |
| 핵심 구조    | NTFS의 중심    |

---

👉 한 줄  
➡️ **“MFT = 파일 시스템 핵심 DB”**

---

#### 📌 6. NTFS 주요 개념

## ✅ ACL (Access Control List)

| 항목  | 설명       |
| --- | -------- |
| ACL | 접근 권한 목록 |
| 구성  | 사용자 + 권한 |

## ✅ 저널링

| 항목  | 설명       |
| --- | -------- |
| 기능  | 변경 내용 기록 |
| 목적  | 장애 복구    |

---

# 🎯 초압축 암기 (시험 직전)

```id="ntfs-final"
NTFS:
ACL / 저널링 / 암호화

EFS:
파일 암호화 (사용자 기반)

BitLocker:
디스크 전체 암호화 ⭐

구조:
Boot → MFT → Data

핵심:
MFT = 파일 정보
```
