# ✅ 1. 서비스 거부 공격 (DoS / DDoS)

## 🔷 DoS 설명

- 특정 시스템 또는 네트워크 자원을 고갈시켜 정상적인 서비스를 불가능하게 만드는 공격
    
- 단일 공격자가 수행
    

---

## 🔷 DDoS 기본 개념

- 다수의 감염된 시스템(좀비 PC)을 이용하여 공격
    
- 공격자는 C&C 서버를 통해 봇을 제어
    

👉 특징

- 탐지 어려움
    
- 대규모 트래픽 발생
    

---

## 🔷 DoS / DDoS 공격 도구

- Trinoo
    
- TFN (Tribe Flood Network)
    
- Stacheldraht
    
- LOIC (Low Orbit Ion Cannon)
    
- HOIC
    

---

## 🔷 공격 유형별 구분

|유형|설명|
|---|---|
|Flooding|트래픽 과부하|
|Protocol Attack|서버 자원 고갈|
|Application Attack|웹 서버 공격|

# ✅ 2. 분산 서비스 거부 공격 (DDoS 유형)

---

## 🔷 TCP SYN Flooding

### ✔ 공격 방법

- TCP 3-way handshake 과정에서 SYN만 보내고 ACK 응답 안함
    
- 서버는 half-open 상태 유지 → 자원 고갈
    

---

### ✔ 대응 방법

- SYN Cookie
    
- Backlog Queue 증가
    
- 방화벽 설정
    

---

### ✔ 라우터 대응

- SYN rate limit
    
- ACL 설정
    
- Anti-Spoofing 적용
    

---

## 🔷 ICMP Flooding

### ✔ 공격 방법

- 대량의 ICMP Echo Request(ping) 전송
    

### ✔ 사용 메시지

- Echo Request
    
- Echo Reply
    

---

### ✔ 대응 방법

- ICMP rate limit
    
- 방화벽 차단
    
- 불필요 ICMP 차단
    

---

## 🔷 Tear Drop / Ping of Death

### ✔ 공격 종류

- IP Fragmentation 공격

> ip 패킷이 네트워크의 MTU 보다 크면 여러 개의 Fragment로 나눠서 전송
> 수신 측에서 이를 다시 재조립


> 조각난 fragment의 offset 값을 비정상적으로 설정
> 패킷들이 겹치거나 순서가 꼬이도록 전송


> 수신 시스템이 재조립 과정에서 오류 발생 or 커널 충돌 시스템 다운



---

### ✔ Ping of Death

- 65535byte 초과 패킷 전송
    
- 시스템 버퍼 오버플로우 유발
    

---

## 🔷 Land Attack

- 출발지 IP = 목적지 IP 동일
    
- 시스템 혼란 유발
    

---

## 🔷 HTTP GET Flooding

- 정상 HTTP 요청처럼 대량 요청
    
- 웹 서버 자원 고갈
    

---

## 🔷 Cache Control 공격

- 캐시 우회 요청 반복
    
- 서버 부하 증가
    

---

## 🔷 Slow HTTP GET/POST

### ✔ 공격 방법

- 연결 유지 후 데이터 천천히 전송
    
- 서버 connection 고갈
    

---

## 🔷 Hash DoS

- 해시 충돌 유도
    
- 서버 CPU 과부하
    

---

## 🔷 Hulk DoS

- HTTP GET Flooding 변형
    
- User-Agent 변조

# ✅ 3. 스캐닝 (Scanning)

## 🔷 nmap 포트 스캐닝 종류

|종류|설명|
|---|---|
|UDP Scan|UDP 포트 확인|

# ✅ 4. 스니핑 공격

## 🔷 개념

- 네트워크 패킷 도청
    

---

## 🔷 특징

- Promiscuous Mode (무차별 모드)
    
- IP, Port, 데이터 확인 가능
    

---

## 🔷 tcpdump

### ✔ 사용법

```
tcpdump -i eth0
```

---

### ✔ 주요 옵션

|옵션|설명|
|---|---|
|-i|인터페이스|
|-n|DNS 해석 안함|

|-w|파일 저장|
|-r|파일 읽기|

---

- 세션 ID 탈취 후 인증 우회
    

---

### ✔ 특징

- 인증 절차 없이 접근 가능
    

---

### ✔ 원인

- 평문 전송
    
- 세션 관리 취약

# ✅ 🔥 시험용 핵심 요약 (초압축)

👉  
DoS는 단일 공격, DDoS는 분산 공격이며  
TCP SYN Flooding은 half-open 연결을 유발하여 자원을 고갈시키고  
ICMP Flooding은 ping 트래픽으로 과부하를 유발한다.

스니핑은 패킷을 도청하는 공격이며  
스푸핑은 IP나 MAC을 위조하는 공격이다.
