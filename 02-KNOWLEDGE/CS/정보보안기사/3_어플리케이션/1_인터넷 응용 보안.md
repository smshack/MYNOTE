
# ✅ 1. FTP

## 🔷 개요

- 파일 전송 프로토콜 (TCP 21번 포트 사용)
    
- 클라이언트 ↔ 서버 간 파일 송수신
    

---

## 🔷 특징

- 제어 연결(Control) + 데이터 연결(Data) 분리
    
- 평문 전송 (보안 취약)
    

---

## 🔷 로그인 과정

1. Control Connection (21번 포트)
    
2. USER / PASS 전달
    
3. 인증 성공
    
4. 데이터 연결 생성
    

---

## 🔷 종류

- Active Mode
    
- Passive Mode
    

---

## 🔷 Active vs Passive

|구분|Active|Passive|
|---|---|---|
|연결|서버 → 클라이언트|클라이언트 → 서버|
|포트|서버 20번|서버 임의 포트|
|특징|방화벽 문제|안전|

---

## 🔷 접근통제 파일

- `/etc/ftpusers` → 로그인 금지 계정
    
- `/etc/vsftpd.user_list`
    

---

## 🔷 로그 기록

- `/var/log/xferlog`
    
- `/var/log/vsftpd.log`
    

---

## 🔷 보안 취약점

- 평문 전송 (계정 탈취)
    
- Anonymous 로그인
    
- Bounce 공격
    

---

## 🔷 보안 대책

- SFTP / FTPS 사용
    
- Anonymous 비활성화
    
- 접근제어 설정

# ✅ 2. E-Mail 보안

---

## 🔷 SMTP

- 메일 송신 (TCP 25)
    

---

## 🔷 POP3

- 메일 수신 (TCP 110)
    
- 다운로드 후 삭제
    

---

## 🔷 IMAP

- 메일 서버에 저장 (TCP 143)
    

---

## 🔷 SMTP 메일 처리 방식

1. 송신자 → SMTP 서버
    
2. SMTP → 수신 서버
    
3. POP3/IMAP으로 수신
    

---

## 🔷 메일 보안 기술

|기술|설명|
|---|---|
|PGP|개인키/공개키 암호화|
|PEM|인증 기반|
|S/MIME|전자서명 + 암호화|

# ✅ 🔥 최종 시험용 요약

👉  
FTP는 파일 전송 프로토콜로 평문 전송의 취약점이 있으며 SFTP로 보완한다.  
SMTP는 메일 송신, POP3/IMAP은 수신 프로토콜이다.

웹 서버는 디렉터리 리스팅 차단, 로그 관리, WAF 적용이 중요하며  
DNS는 도메인 변환 시스템으로 DNSSEC을 통해 무결성을 보장한다.
