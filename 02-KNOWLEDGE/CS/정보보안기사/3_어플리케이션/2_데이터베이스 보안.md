# ✅ 3. MySQL

---

## 🔷 1) DB 연결

```id="mysql-connect"
mysql -u root -p
```

---

## 🔷 2) 사용자 생성

```id="mysql-user"
CREATE USER 'user'@'%' IDENTIFIED BY 'password';
```

---

## 🔷 3) 권한 부여 / 회수

### ✔ GRANT

```id="mysql-grant"
GRANT ALL PRIVILEGES ON db.* TO 'user'@'%';
```

---

### ✔ REVOKE

```id="mysql-revoke"
REVOKE ALL PRIVILEGES ON db.* FROM 'user'@'%';
```

# ✅ 5. 데이터베이스 보안

---

## 🔷 개요

- 데이터의 **기밀성, 무결성, 가용성(CIA)** 확보
    

---

## 🔷 보안 위협 요소

### ✔ 집합성 (Aggregation)

- 낮은 등급 데이터 결합 → 높은 정보 노출
    

---

### ✔ 추론 (Inference)

- 간접 정보로 민감 데이터 추정
    

---

## 🔷 보안 요구사항

|항목|설명|
|---|---|
|기밀성|접근 제한|
|무결성|데이터 보호|
|가용성|서비스 유지|
|인증|사용자 확인|
|접근통제|권한 관리|

---

## 🔷 MySQL 백업

### ✔ 백업

```id="mysqldump-backup"
mysqldump -u root -p dbname > backup.sql
```

---

### ✔ 복구

```id="mysqldump-restore"
mysql -u root -p dbname < backup.sql
```

# ✅ 🔥 시험용 핵심 요약

👉  
DBMS는 데이터를 효율적으로 관리하는 시스템이며  
보안 측면에서는 기밀성, 무결성, 가용성을 확보해야 한다.

집합성과 추론 공격은 데이터 유출 위험이 있으며  
이를 방지하기 위해 암호화, 접근통제, 감사 기능을 적용한다.
