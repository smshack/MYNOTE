# ✅ 3. SSL (TLS)

## 🔷 개요

- 웹 통신 암호화 프로토콜 (HTTPS)
- 개방환경에서 송수신 간 안전한 통신을 위해 개발된 암호 전송 프로토콜
- RSA 알고리즘 사용 443 포트 사용
    

---

## 🔷 보안 서비스

- 기밀성: 데이터를 불법적인 접근을 통제 
    
- 무결성: 송수신 간 위변조 방지
    
- 인증: 신뢰할 수 있으며 검증된 사이트인지 상대 사이트 인증
    

---

## 🔷 Handshake 과정

```id="ssl-flow"
1. Client Hello
2. Server Hello
3. 인증서 전달
4. 키 교환
5. 세션키 생성
6. 암호화 통신 시작
```

---

## 🔷 세부 과정

- Cipher Suite 협상
    
- 공개키로 대칭키 교환
    
- 세션키 생성
    
---

| 계층  | 프로토콜                            | 역할      | 주요 기능                         | 특징           |
| --- | ------------------------------- | ------- | ----------------------------- | ------------ |
| 상위  | Application (HTTP 등)            | 사용자 데이터 | 웹 서비스 제공                      | SSL 위에서 동작   |
| SSL | **Handshake Protocol**          | 보안 협상   | 인증서 교환, 키 교환, Cipher Suite 협상 | 연결 초기 1회 수행  |
| SSL | **Change Cipher Spec Protocol** | 암호화 전환  | 암호화 적용 알림                     | 단순 신호 프로토콜   |
| SSL | **Alert Protocol**              | 오류 처리   | 경고, 연결 종료                     | 보안 이벤트 전달    |
| SSL | **Record Protocol** ⭐           | 데이터 보호  | 분할, 압축, MAC, 암호화              | 실제 데이터 전송 핵심 |
| 하위  | TCP                             | 전송      | 신뢰성 있는 데이터 전달                 | SSL 하위 계층    |

---

Handshake로 키를 교환하고,  
Change Cipher로 암호화를 적용한 뒤,  
Record Protocol을 통해 데이터를 암호화 전송하며,  
Alert Protocol로 오류를 처리한다.

---

SSL은 Handshake, Record, Change Cipher Spec, Alert Protocol로 구성되며  
Handshake를 통해 인증과 키 교환을 수행하고 Record Protocol을 통해 데이터를 암호화하여 전송한다.

---

## 🔷 OpenSSL 사용

```id="openssl"
openssl s_client -connect example.com:443
```

---

## 🔷 취약점

| 취약점            | 대상      | 공격 원리                    | 공격 방식               | 영향         | 대응 방법                |
| -------------- | ------- | ------------------------ | ------------------- | ---------- | -------------------- |
| **Heartbleed** | OpenSSL | Heartbeat 기능 오류 (메모리 반환) | 요청 길이를 속여 서버 메모리 유출 | 개인정보, 키 유출 | OpenSSL 패치, 키 재발급    |
| **SSL Strip**  | HTTPS   | HTTPS → HTTP 다운그레이드      | 중간자(MITM)가 암호화 제거   | 평문 통신 노출   | HSTS 적용, HTTPS 강제    |
| **POODLE**     | SSL 3.0 | CBC 패딩 취약점               | 암호문 일부 조작하여 복호화     | 쿠키 탈취      | SSL 3.0 비활성화, TLS 사용 |

# ✅ 6. OTP

## 🔷 개요

- 매번 다른 난수를 생성하여 인증시 패스워드를 대신하는 방법
    

---

## 🔷 방식

| 방식   | 설명     |                                                                                            |
| ---- | ------ | ------------------------------------------------------------------------------------------ |
| 동기식  | 시간 기반  | 사용자의 OTP 생성 매체와 은행의 인증서버 사이에 동기화된 기준 값에 따라 otp가 생성                                         |
| 비동기식 | 이벤트 기반 | 질의 응답 사용자의 생성 매체와 은행의 인증서버 사이에 동기화되는 기준 값이 없으며 사용자가 직접 임의의 낮수를 otp 생성 매체에 입력하여 otp를 생성하는 방 |
