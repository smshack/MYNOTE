## ✔ 5. Kerberos 인증 

## ✔ 개념

- **대칭키 기반 중앙 인증 시스템**
    
- 티켓 기반 인증
    

---

## ✔ 구성 요소

|요소|설명|
|---|---|
|KDC|키 배포 센터|
|AS|인증 서버|
|TGS|티켓 발급 서버|
|Principal|사용자|
|Ticket|인증 정보|

---

## ✔ 동작 원리 (중요 ⭐)

```id="bczn8p"
1. 사용자 → AS : 로그인 요청
2. AS → 사용자 : TGT 발급
3. 사용자 → TGS : 서비스 요청 + TGT
4. TGS → 사용자 : 서비스 티켓 발급
5. 사용자 → 서버 : 티켓 전달 → 인증 완료
```

#### 악성코드

| 유형                   | 정의                       | 특징            | 대응 방법          |
| -------------------- | ------------------------ | ------------- | -------------- |
| 논리폭탄 (Logic Bomb)    | 특정 조건에서 실행되는 악성 코드       | 시간/이벤트 조건 필요  | 코드 검증, 변경관리    |
| 키로거 (Keylogger)      | 키보드 입력을 기록하는 악성 코드       | 계정/비밀번호 탈취    | 백신, 보안 키보드     |
| 트로이목마 (Trojan Horse) | 정상 프로그램으로 위장한 악성 코드      | 사용자 실행 필요     | 출처 검증, 실행 제한   |
| 백도어 (Backdoor)       | 인증 없이 시스템 접근 가능하게 하는 코드  | 원격 제어 가능      | 접근 통제, 로그 모니터링 |
| 루트킷 (Rootkit)        | 시스템 깊숙이 숨어 권한 유지하는 악성 코드 | 은닉 기능, 탐지 어려움 | 무결성 검사, OS 재설치 |
