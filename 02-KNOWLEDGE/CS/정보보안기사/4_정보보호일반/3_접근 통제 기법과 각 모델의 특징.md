
# 🔐 1. 접근 통제 방식

## 1) Capability List (C-List)

- **주체 중심**
    
- 각 주체가 접근 가능한 객체와 권한을 리스트로 보유
    

👉 특징

- 권한 확인 빠름 (주체 기준)
    
- 권한 회수 어려움 ❌
    

---

## 2) Access Control List (ACL)

- **객체 중심**
    
- 각 객체가 어떤 주체에게 어떤 권한을 주는지 관리
    

👉 특징

- 권한 관리 쉬움 (객체 기준)
    
- 파일 시스템에서 많이 사용
    

---

# 📊 2. 접근 통제 매트릭스

👉 개념

- 행: 주체 (User)
    
- 열: 객체 (Resource)
    
- 값: 권한 (Read, Write 등)
    

---

## 종류

### 1) ACL

- 객체 기준 접근 제어
    

### 2) Capability List

- 주체 기준 접근 제어
    

### 3) 내용 기반 접근 통제 (Content-Based)

- 데이터 내용에 따라 접근 제어  
    👉 예: DB에서 “부서=인사팀만 조회 가능”
    

### 4) 제한적 인터페이스

- 권한 없으면 기능 자체 접근 불가  
    👉 UI/API 차단
    

---

# 🧠 3. 접근 통제 모델

## 1) Bell-LaPadula 모델 (기밀성)

👉 “정보 유출 방지 모델”

- ❌ No Read Up (NRU) → 위는 못 읽음
    
- ❌ No Write Down (NWD) → 아래로 못 씀
    

👉 핵심  
👉 **기밀성 보호**

---

## 2) Biba 모델 (무결성)

👉 Bell-LaPadula 반대

- ❌ No Read Down (NRD) → 아래는 못 읽음
    
- ❌ No Write Up (NWU) → 위로 못 씀
    

👉 핵심  
👉 **데이터 신뢰성 보호**

---

## 3) Clark-Wilson 모델 (상업용 무결성)

- 프로그램(트랜잭션) 통해서만 접근 가능
    
- 직접 접근 금지
    

👉 특징

- 금융/회계 시스템에 적합
    
- 무결성 + 감사성 강조
    

---

# 🖥️ 4. 보안 운영체제 (Secure OS)

👉 정의  
기존 OS에 보안 기능 강화

## 구조 종류

- 신뢰 필터 구조
    
- 커널 구조
    
- 이중 커널 구조
    
- 중복 구조
    

👉 핵심  
👉 **커널 레벨에서 접근 통제 수행**

# 🔑 5. 키 분배 및 암호화

---

## 1) 대칭키 암호화

👉 특징

- 하나의 키 사용
    
- 빠름 ⚡
    
- 키 분배 문제 존재 ❗
    

---

### 🔹 스트림 암호

- 1bit 또는 1byte 단위 암호화
    
- 빠름, 실시간 처리에 적합  
    👉 예: RC4
    

---

### 🔹 블록 암호

#### ✔ 개념

- 일정 크기 블록 단위 암호화
    

#### ✔ 방식

- ECB, CBC, CFB, OFB 등
    

#### ✔ 장점

- 보안성 높음
    
- 데이터 무결성 강화
    

#### ✔ 종류

- AES
    
- DES
    
- SEED
    

---

## 2) 공개키 암호화 (비대칭키)

👉 특징

- 공개키 + 개인키 사용
    
- 키 분배 문제 해결
    
- 속도 느림 ❌
    

---

### 대표 알고리즘

- Diffie-Hellman
    
- RSA
    
- DSA
    
- Elliptic Curve Cryptography
    

---

# ⚔️ 6. 대칭키 vs 공개키 차이

|구분|대칭키|공개키|
|---|---|---|
|키 개수|1개|2개|
|속도|빠름|느림|
|키 분배|어려움 ❌|쉬움 ⭕|
|보안성|중간|높음|
|용도|데이터 암호화|키 교환 / 인증|

---

# 💡 실무 핵심 (중요)

👉 실제 시스템은 이렇게 사용됨

1. 공개키로 **대칭키 전달**
    
2. 이후는 대칭키로 통신
    

👉 예:

- HTTPS
    
- VPN
