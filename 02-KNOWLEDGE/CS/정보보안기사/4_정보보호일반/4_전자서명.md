# 📘 1. 전자서명의 이해

## 📌 정의

- 전자문서에 대해 **작성자 확인 + 위변조 방지**를 위해 사용하는 암호 기술  
    👉 공개키 암호 방식 기반
    

---

## 📌 특징

|특징|설명|
|---|---|
|서명자 인증|송신자 신원 확인|
|부인 방지|서명자가 부인 불가|
|위조 불가|타인이 서명 생성 불가|
|변경 불가|내용 변경 시 검증 실패|
|재사용 불가|다른 문서에 재사용 불가|

---

## 📌 기능

|기능|설명|
|---|---|
|인증|송신자 확인|
|무결성|데이터 변경 여부 확인|
|부인방지|송신자가 행위 부인 불가|

---

# 📘 2. 전자서명 과정 🔥

## 📌 과정 순서

1. 메시지 작성
    
2. 해시 함수 적용 → 메시지 다이제스트 생성
    
3. **개인키로 암호화 → 전자서명 생성**
    
4. 메시지 + 서명 전송
    

---

## 📌 검증 과정

1. 수신자가 메시지 수신
    
2. 동일 해시 함수 적용
    
3. 송신자의 **공개키로 서명 복호화**
    
4. 해시값 비교 → 동일하면 정상
    

---

# 📘 3. 전자서명 기법 종류

| 알고리즘    | 특징                       |
| ------- | ------------------------ |
| RSA     | 가장 대표적 공개키 기반            |
| ElGamal | 이산 로그 기반                 |
| Schnorr | 효율성 높음 + IC 카드 + 서명크기 작음 |
| DSS     | 미국 표준 전자서명+ 오직 전자서명만     |
| KCDSA   | 한국형 전자서명                 |
| ECC     | 짧은 키로 높은 보안성             |

# 📘 4. PKI (Public Key Infrastructure)

## 📌 개념

- 공개키 기반으로 **인증서 발급 및 관리하는 인프라**
    

👉 은행, 증권, 보험 등 공인인증 구조

---

## 📌 목적

|목적|설명|
|---|---|
|인증|사용자 신원 확인|
|무결성|데이터 위변조 방지|
|부인방지|거래 부인 방지|
|기밀성|암호화 지원|

---

## 📌 구성요소

|구성|설명|
|---|---|
|CA (인증기관)|인증서 발급|
|RA (등록기관)|사용자 신원 확인|
|CRL|폐기된 인증서 목록|
|Directory|인증서 저장소|
|CPS|인증서 정책 문서|
|X.509|인증서 표준 형식|

---

## 📌 세부 설명

- **CA**: 인증서 발급 및 서명
    
- **RA**: 사용자 인증 후 CA에 요청
    
- **CRL**: 폐기된 인증서 리스트
    
- **Directory**: 인증서 공개 저장
    
- **CPS**: 인증 정책 정의
    
- **X.509**: 인증서 구조 표준
    

---

# 📘 5. PMI (Privilege Management Infrastructure)

## 📌 개념

- **권한(Privilege) 관리 인프라**  
    👉 PKI가 “신원 인증”이라면  
    👉 PMI는 “권한 관리”
    

---

## 📌 속성 인증서 (Attribute Certificate)

- 사용자 권한 정보를 담은 인증서
    
- 신원(X.509)과 별도 관리
    

---

## 📌 구성요소

|구성|설명|
|---|---|
|SOA|권한 정책 최상위 기관|
|AA|속성 인증서 발급|
|권한 소유자|사용자|
|권한 검증자|권한 확인 주체|

---

## 📌 세부 설명

- **SOA (Source of Authority)**: 권한 정책 정의
    
- **AA (Attribute Authority)**: 권한 인증서 발급
    
- **권한 소유자**: 사용자
    
- **권한 검증자**: 서비스 제공자
    

---

# 📘 6. 전자서명 응용 (전자투표)

## 📌 개념

- 전자적으로 투표를 수행하고 개표하는 시스템
    

---

## 📌 시스템 구성요소

|구성|설명|
|---|---|
|전자 투표기|투표 입력 장치|
|개표 시스템|투표 결과 집계|
|전자 검표 시스템|결과 검증|

---

## 📌 전자투표 요구사항 🔥

|요구사항|설명|
|---|---|
|정확성|투표 결과 정확|
|비밀성|투표 내용 비공개|
|무결성|위변조 방지|
|인증성|유권자 확인|
|부인방지|투표 부인 불가|
|공정성|중간 결과 노출 방지|
|검증성|결과 검증 가능|

---

# 💡 보완 핵심 포인트 (시험 필수)

### 1️⃣ 전자서명 vs 암호화 차이

- 전자서명 → **개인키로 암호화 / 공개키로 검증**
    
- 암호화 → **공개키로 암호화 / 개인키로 복호화**
    

---

### 2️⃣ PKI vs PMI

|구분|PKI|PMI|
|---|---|---|
|목적|신원 인증|권한 관리|
|인증서|공개키 인증서|속성 인증서|

---

### 3️⃣ 전자서명 핵심 3요소

👉 인증 / 무결성 / 부인방지
