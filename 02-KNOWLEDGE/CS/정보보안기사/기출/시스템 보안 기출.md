### 운영제체의 목적
1. **처리능력의 향상**: 단위 시간당 처리하는 작업량
2. **신뢰도 향상**: 하드웨어, 소프트웨어가 실패 없이 주어진 기능을 수행할 수 있는 능력
3. **응답시간 단축**:사용자가 시스템에 작업을 의뢰한 후 반응을 얻을 때까지의 시간
4. **반환시간 최소화**: 하나의 작업이 시작한 시간에서 부터 결과를 얻을 때까지 걸리는 시간
5. **사용 가능도 향상**: 가동률이라고 하고, 시스템을 사용할 필요가 있을 때 얼마나 빨리 사용할 수 있는가

---
### 운영체제 동작 원리
- **듀얼모드**: 운영체제의 구조는 이중모드로 되어 있고 이는 **사용자 모드**와 **커널모드**로 되어 있음

---
### 커널
- 운영체제의 일부분으로 **하드웨어의 관리**를 담당
- 유닉스 계열의 시스템이 부팅될 때 가장 먼저 읽혀지는 운영제체의 **주기억장치**에 상주함

---
### 시분할 시스템
- 여러 명의 사용자가 사용하는 시스템에서 아용자들의 프로그램을 번갈아거며 처리함으로 , 각 사용자는 독립된 컴퓨터를 사용하는 느낌을 주는 것으로 라운드로빈 방식
- 아주 짧은 일정 시간 동안만 CPU를 사용하고 다음 사용자에게로 사용 권한을 전환, 각 사용자는 자신이 컴퓨터 시스템을 독점하고 있는 것 처럼 생각
---
### 임베디드 시스템
- 어떤 기계 안에 제어를 목적으로 하는 아주 작은 컴퓨터
- 다른 하드웨어나 소프트웨어의 일부로 내포되어 있는 내장형 의미
- 각종 전자제품과 정보기기 등에 설치된 시스템
---
### 버퍼링 & 스풀링
- **버퍼링**: 외부자료가 주기억장치(RAM)에 저장
> 데이터를 전송할 때 양자의 속도차를 수정하기 위해 중간에서 데이터를 일시적으로 기억장소에 축적하는 방법

- **스풀링**: 외부 자료가 보조기억장치(HDD)에 저장
> 처리 지연을 단축하기 위해 보조기억장치를 완충 기억장치로 사용하는 것

---

| 구분     | 버퍼링 (Buffering)        | 스풀링 (Spooling)        |
| ------ | ---------------------- | --------------------- |
| 구현 방식  | 하드웨어적 구현               | 소프트웨어적 구현             |
| 위치     | 주기억장치 (메모리)            | 디스크                   |
| 사용자 환경 | Single User            | Multi User            |
| 수행 방식  | 스택 또는 큐 방식 입출력 수행      | 큐 방식 입출력 수행           |
| 다중 작업  | 한 작업 내에서 입출력과 계산 중복 가능 | 여러 작업 간 입출력과 계산 중복 가능 |

### 프로세스
- 실행 중인 프로그램
- 프로시저가 활동 중인것
- 비동기적인 행위를 일으키는 주체
- PCB(Processor control block)의 존재로서 명시되는 것

---

| 구성 요소                       | 설명                                                   |
| --------------------------- | ---------------------------------------------------- |
| PCB (Process Control Block) | 프로세스의 상태 정보를 저장하는 구조체 (PID, 상태, 레지스터, 프로그램 카운터 등 포함) |

### 프로세스 상태 전이

![[Pasted image 20260408213520.png]]


**프로세스는 Ready → Running → Block 상태를 반복하며 CPU를 사용**

---

| 구분  | 선점(Preemptive) 스케줄링                                        | 비선점(Non-preemptive) 스케줄링                              |
| --- | ---------------------------------------------------------- | ----------------------------------------------------- |
| 개념  | 한 프로세스가 CPU를 사용 중일 때, 더 높은 우선순위 프로세스가 현재 작업을 중단시키고 CPU를 점유 | 한 프로세스가 CPU를 할당받으면 작업 종료 또는 반환 시까지 다른 프로세스가 CPU 점유 불가 |
| 특징  | 높은 우선순위 프로세스가 빠르게 처리됨                                      | 한 번 시작된 작업은 끝까지 실행                                    |
| 장점  | - 응답속도 빠름- 대화형/시분할 시스템에 적합                                 | - 응답시간 예측 용이- 공정한 처리 가능                               |
| 단점  | - 문맥교환(Context Switch)으로 오버헤드 발생- 우선순위 높은 작업 많으면 비효율       | - 긴 작업이 CPU 점유 시 짧은 작업 대기- 응답성 낮음                     |

---
#### 비선점형 스케줄링 알고리즘
> 각 프로세스가 제어권을 따로 가짐(프로세스를 강제로 끄집어 낼 수 없다) 

| 알고리즘                                  | 설명                                                         |
| ------------------------------------- | ---------------------------------------------------------- |
| **FIFO (First In First Out)**         | 가장 먼저 도착한 프로세스부터 CPU를 할당하는 가장 단순한 방식                       |
| **SJF (Shortest Job First)- 최소작업 우선** | 실행 시간이 가장 짧은 작업을 먼저 수행하는 방식 (기아현상 발생 가능)                   |
| **HRN (Highest Response Ratio Next)** | 응답률 = (대기시간 + 서비스시간) / 서비스시간으로 계산하여 가장 높은 값부터 실행 (기아현상 완화) |

---
### 선점형 스케줄링 알고리즘
>  운영체제가 제어권을 강력하게 선점(프로세스를 강제로 끄집어 낼 수 있음)

| 알고리즘                                 | 설명                                                       |
| ------------------------------------ | -------------------------------------------------------- |
| **Priority (Priority Scheduling)**   | 각 프로세스에 우선순위를 부여하여 높은 순서대로 실행 (선점/비선점 모두 가능, 기아현상 발생 가능) |
| **SRT (Shortest Remaining Time)**    | 남은 실행 시간이 가장 짧은 프로세스를 먼저 수행하는 방식 (SJF의 선점형, 기아현상 발생)     |
| **RR (Round Robin)**                 | 각 프로세스에 동일한 CPU 시간(Time Quantum)을 할당하여 순환하며 실행           |
| **MLQ (Multi-Level Queue)- 다단계 큐**   | 프로세스를 여러 큐로 분리하여 각 큐마다 다른 스케줄링 적용 (큐 간 이동 없음, 기아현상 발생)   |
| **MFQ (Multi-Level Feedback Queue)** | 프로세스가 큐 간 이동 가능, 실행 패턴에 따라 우선순위 동적으로 변경                  |
| **Deadline (Deadline Scheduling)**   | 프로세스마다 처리 기한(Deadline)을 부여하여 기한 내 수행하도록 스케줄링             |

 - **프로세스 동기화 (Process Synchronization)**
>  프로세스에 대한 처리 순서를 결정

| 항목  | 내용                                   |
| --- | ------------------------------------ |
| 개념  | 여러 프로세스/스레드가 공유 자원에 올바르게 접근하도록 순서 제어 |
| 목적  | 데이터 일관성 유지, Race Condition 방지        |
| 문제  | 동시에 접근 시 데이터 불일치 발생                  |
| 해결  | 임계 영역 + 상호 배제 적용                     |

#### 📌 2. 임계 영역 (Critical Section)
> 비동기 절차에 있어서 자기 자신이 또는 다른 비동기 절차 부분과 동시에는 실행할 수 없는 부분
> 하나의 프로세스가 공유 데이터에 접근하면, 다른 프로세스는 그 공유 데이터에 접근해서는 안됨

| 항목  | 내용                |
| --- | ----------------- |
| 개념  | 공유 자원에 접근하는 코드 영역 |
| 문제  | 동시에 실행 시 오류 발생    |

#### 🔥 해결 조건

| 조건                       | 설명               |
| ------------------------ | ---------------- |
| 상호 배제 (Mutual Exclusion) | 한 번에 하나만 접근      |
| 진행 (Progress)            | 다음 실행 프로세스 결정 가능 |
| 한정 대기 (Bounded Waiting)  | 무한 대기 방지         |

## 📌 3. 상호 배제 기법 (Mutual Exclusion)
> 임의의 시점에서 하나의 프로세스만 그 접근을 허용

| 기법           | 설명                |
| ------------ | ----------------- |
| Lock (Mutex) | 자원에 잠금 설정         |
| Semaphore    | 정수값으로 자원 접근 제어    |
| Monitor      | 고급 동기화 구조 (언어 지원) |

---
### 🔹 세마포어 종류
> 운영체제 또는 프로그램 작성 내에서 공유 자원에 대한 접속을 제어하기 위해 사용되는 신호

| 종류                 | 설명                |
| ------------------ | ----------------- |
| Binary Semaphore   | 0/1 값 (Mutex와 유사) |
| Counting Semaphore | 여러 자원 개수 관리       |
### 모니터
> 프로세스 또는 스레드를 동기화 하는 방법 중 하나, 한번에 하나의 프로세스만 모니터에서 활동하도록 보장

## 📌 4. 교착상태 (Deadlock)

| 항목  | 내용                          |
| --- | --------------------------- |
| 개념  | 프로세스들이 서로 자원을 기다리며 무한 대기 상태 |

##### 🔥 발생 조건 (4가지 모두 만족 시 발생)

| 조건        | 설명              |
| --------- | --------------- |
| **상호 배제** | 자원을 하나씩만 사용     |
| **점유 대기** | 자원 보유 + 추가 요청   |
| **비선점**   | 강제로 자원 회수 불가    |
| **순환 대기** | 프로세스 간 자원 대기 순환 |

---

### 🔹 해결 방법

| 방법                  | 설명                  |

| **회피 (Avoidance)**  | 안전 상태 유지 (은행가 알고리즘) |
| **탐지 (Detection)**  | 발생 후 탐지             |
| **회복 (Recovery)**   | 프로세스 종료 / 자원 회수     |

### 기억장치의 관리 기법
#### 단편화
> 분할된 주 기억장치에 프로그램을 할당하고 반납하는 과정을 반복하면서 사용되지 않고 남는 기억장치의 빈 공간이 발생

> 연속으로 기억장치를 할당하여 사용되지 못하는 공간이 생길 수 있음(단편화 공간)

---
### 내부 단편화
> 분할된 영역에 프로그램을 사용하고 남은 빈공간
### 외부 단편화
> 분할된 영역이 할당될 프로그램의 크기보다 작기 때문에 프로그램이 할당 될 수 없어 사용되지 않는 빈 공간

### 디스크 접근 시간

| 항목                              | 설명                                   |
| ------------------------------- | ------------------------------------ |
| 탐색시간 (Seek Time)                | 디스크 헤드가 목표 트랙(실린더)까지 이동하는 데 걸리는 시간   |

| 전송시간 (Transfer Time)            | 실제 데이터를 디스크에서 메모리로 전송하는 데 걸리는 시간     |

### 디스크 스케줄링 종류
![[Pasted image 20260408232411.png]]

|알고리즘|설명|
|---|---|
|FCFS (First Come First Serve)|먼저 요청된 순서대로 디스크를 처리하는 방식 (구현 간단, 비효율 가능)|
|SSTF (Shortest Seek Time First)|현재 헤드 위치에서 가장 가까운 요청을 먼저 처리하는 방식 (평균 탐색 시간 감소, 기아현상 발생 가능)|
|SCAN (Elevator Algorithm)|디스크 헤드가 한 방향으로 이동하며 요청을 처리하고 끝에 도달하면 반대 방향으로 이동 (엘리베이터 방식)|
|C-SCAN (Circular SCAN)|한 방향으로만 이동하며 요청 처리 후, 끝에 도달하면 처음으로 이동하여 다시 처리 (균등한 응답 시간 제공)|
|C-LOOK (Circular LOOK)|요청이 있는 범위까지만 이동 후 처음 위치로 돌아와 다시 처리 (불필요한 이동 감소)|-

#### RAID

> 데이터를 분할해서 복수의 자기 디스크 장치에 대해 병렬로 데이터를 저장하고 읽는 방식
>

> 여러 디스크를 병렬로 연결하여 사용하는 기법으로 접근과 속도 데이터 보존 신뢰가 우수할 수록 높은 등급을 받는 6단계로 구분

---

| RAID 레벨                                            | 설명                                         |
| -------------------------------------------------- | ------------------------------------------ |
| RAID 0 (Striping)                                  | 데이터를 여러 디스크에 분산 저장하여 속도 향상 (장애 대비 없음)      |
| RAID 1 (Mirroring)                                 | 동일 데이터를 두 개 디스크에 복제 저장 (안정성 높음, 용량 50% 사용) |
| RAID 2 (Bit-level Striping + Hamming Code)         | 비트 단위 분산 + 오류 검출/정정 코드 사용 (거의 사용 안 함)      |
| RAID 3 (Byte-level Striping + Parity)              | 바이트 단위 분산 + 패리티 디스크 1개 (연속 데이터 처리에 유리)     |
| RAID 4 (Block-level Striping + Parity)             | 블록 단위 분산 + 패리티(에러제어) 디스크 1개 (패리티 병목 발생)    |
| RAID 5 (Block-level Striping + Distributed Parity) | 블록 단위 분산 + 패리티를 여러 디스크에 분산 (성능 + 안정성 균형)   |
| RAID 6 (Block-level Striping + Dual Parity)        | RAID 5 + 이중 패리티 (2개 디스크 장애 허용)             |

### 패리티 검사
> 데이터의 저장과 전송의 정확성을 유지하기 위해 검사 비트를 이용하는 자동 오류 검사 방법


> 데이터 한 블록 끝에 1비트의 검사 비트인 패리티 비트를 추가하여 전송 에러를 검출하는 방법

### 해밍코드
- 컴퓨터 스스로 데이터의 오류를 검출하고 수정하는 오류 수정 코드
- 송수신 데이터의 각 대응하는 비트가 서로 다른 비트의 수를 해밍 거리라고 한다
- 자기 정정 부호의 하나로 비트 착오를 검출해서 오류의 발견은 물론이고 교정까지 가능

### TMS(Threat Management System) 위협 관리 시스템

- 전사적 IT 인프라에 대한 위협 정보들을 수집, 분석, 경보, 관리하는 정보보호 통합관리 시스템
- SIEM 에서 위협 식별 후 TMS에 쿼리를 요청하여 공격주체 식별이 가능

### BitLocker(비트로커)

- 암호화 키 생성 저장
- 무결성 검증을 위한 측정값 저장
- 디지털 인증서 관련 신뢰 연산 제공

### 악성코드 분석
- 어던 코드가 악성 코드인지 판별하는 것 부터 그 코드가 어떤 악성 행위를 하는지 파악하고 차단하기 위한 분석 행위


#### 정적 분석
- 프로그램을 실행시키지 않고 분석
- 소스코드, 파일 헤더, 매니페스트, 프로그램 구조 분석

#### 동적 분석
- 실제로 실행시켜 보고 분석
- 입력값을 변화에 따른 결과값을 분석
- 코드가 동작할 수 있는 환경을 조성하는데 어려울수 있음

#### NTFS

---

|구분|FAT12|FAT16|FAT32|NTFS (New Technology File System)|
|---|---|---|---|---|
|최대 볼륨 크기|16MB|2GB|2TB|16EB|

| **하이브**                        | 설명                                                       |
| -------------------------- | -------------------------------------------------------- |
| **HKEY_CLASSES_ROOT (HKCR)**   | 파일 확장자와 연결된 프로그램 정보 및 OLE 객체 클래스 정보 저장 (예: txt → 메모장)    |

| **HKEY_CURRENT_CONFIG (HKCC)** | 현재 시스템에서 사용 중인 하드웨어 프로파일 정보 (부팅 시 생성, 일시적)               |
| **HKEY_PERFORMANCE_DATA**      | 시스템 성능 데이터 제공 (레지스트리 편집기에는 보이지 않음, API로 조회)              |
| **HKEY_DYN_DATA**              | Windows 9x 계열에서 사용, 플러그앤플레이 및 동적 하드웨어 정보 저장 (현재는 사용 안 함) |

### ndd(설정 파라미터 설정)
- 커널 드라이버의 파라미너 정보를 출력하거나 설정
- tcp/ip 프로토콜 관련 드라이버만 지원

```
ndd -set /dev/ip ip_forward_directed_broadcasts 0 
# 스머프(icmp flooding)공격에 대응하기 위해 directed 브로드키스트 아이피 캐핏 포워딩 허용 불가
ndd -set /dev/tcp tcp_conn_req_max_q0 1024
## synflooding에 대응하기 위해 tcp 연결 요청 대기큐의 크기를 1024로 늘림
```

#### icmp(ping)
- 정상적이면 송수신자의 상태 확인 프로토콜
- 송신자 아이피에 공격대상 아이피 세팅 수신자는 임의의 호스트 공격 대상은 자원을 소모
- 스머프 공격(icmp syn flooding) 은 송신자의 아이피를 공격대상 수신자 아이피를 브로드케스트로 세팅 -> 네트워크의 모든 다른 호스트가 공격대상으로 icmp로 보냄

|로그 분석 (Analysis)|타임스탬프, IP, 이벤트 등을 기준으로 로그 간 연관성 분석|

### 카나리 단어 기법 동작 방식
- 스택 가드라고도 함, 메모리 상에서 프로그램의 복귀 주소와 변수/ 버퍼 사이에 특정값(카나리) 값을 저장해두는 기법
- 버퍼오버플로우 발생 시 특정 값의 변조가 발생하므로 이를 탐지하여 차단

### ASLR

- 실행 시 마다 메모리 주소를 변경 시켜 버퍼 오버플로우를 통한 특정 주소 호출을 방지

- 자원 사용에 대한 경쟁을 이용하여 시스템 관리자의 권한을 획득하고, 파일에 대한 접근을 가능하게 하는 공격
