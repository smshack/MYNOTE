## ✅  GitLab 비밀번호 초기화 방법

GitLab 루트 계정 비밀번호를 초기화하려면 다음 방법을 사용하세요.

#### 1️⃣ GitLab Rails 콘솔 접속

```bash
sudo gitlab-rails console
```

#### 2️⃣ 루트 계정 찾기

```ruby
user = User.find_by(username: 'root')
```

#### 3️⃣ 비밀번호 변경

```ruby
user.password = '새로운비밀번호'
user.password_confirmation = '새로운비밀번호'
user.save!
```

#### 4️⃣ 콘솔 종료

```ruby
exit
```

## ✅  GitLab 프로젝트 구조 잡기
```
practice_team/                   ← 조직(회사, 팀) 단위
├── frontend/                 ← 프론트엔드 프로젝트 그룹
│   └── portfolio-frontend
├── backend/                  ← 백엔드 프로젝트 그룹
│   └── portfolio-backend
├── devops/                   ← 인프라/배포 관련
│   ├── jenkins-pipelines
│   ├── docker-images
│   └── helm-charts
├── qa/                       ← 테스트 및 품질 관리
│   └── sonar-reports
└── docs/                     ← 문서나 공통 템플릿
    └── api-specs
```

## 🧭 단계별 설정 방법

### 1️⃣ `practice_team` 그룹 만들기

1. GitLab 웹에서 상단 메뉴 → **“Groups > Your Groups”** 클릭
    
2. **New group** 버튼 클릭
    
3. **Group name**: `practice_team`
    
4. **Visibility**: 필요에 따라 설정 (Private/ Internal/ Public)
    
5. **Create group** 클릭
    

![[깃랩 그룹 생성.png]]
---

### 2️⃣ 하위 그룹 만들기 (frontend, backend, devops, qa, docs)

1. `practice-team` 그룹 페이지로 이동
    
2. 우측 상단 **New subgroup** 클릭
    
3. 예:
    
    - **Subgroup name** → `frontend`
        
    - 이후 `backend`, `devops`, `qa`, `docs`도 같은 방식으로 생성

![[깃랩 서브그룹 생성.png]]
---

### 3️⃣ 각 하위 그룹 안에 프로젝트 만들기

예를 들어 `portfolio-frontend` 만들려면:

1. `practice-team/frontend` 그룹으로 이동
    
2. **New project > Create blank project** 클릭
    
3. **Project name**: `portfolio-frontend`
    
4. Visibility 및 초기화 옵션 설정 후 **Create project** 클릭
    
![[깃랩 프로젝트 생성.png]]
---

### 4️⃣ 결과 경로 확인

이제 각 프로젝트의 경로는 다음처럼 됩니다 👇

| 그룹       | 프로젝트               | Git 주소                                                             |
| -------- | ------------------ | ------------------------------------------------------------------ |
| frontend | portfolio-frontend | `https://gitlab.com/practice-team/frontend/portfolio-frontend.git` |
| backend  | portfolio-backend  | `https://gitlab.com/practice-team/backend/portfolio-backend.git`   |
| devops   | jenkins-pipelines  | `https://gitlab.com/practice-team/devops/jenkins-pipelines.git`    |
| docs     | api-specs          | `https://gitlab.com/practice-team/docs/api-specs.git`              |

---

### 5️⃣ 추가 팁

- CI/CD 공통 스크립트를 `devops/jenkins-pipelines` 그룹에서 관리하면 효율적입니다.
    
- 공통 문서(`README`, `api-specs`)는 `docs` 그룹에 두면 버전 관리에 좋습니다.
    
- 각 프로젝트의 **Group-level variables** (예: Docker registry, SonarQube 토큰)을 상위 그룹(`practice-team`)에 설정하면 하위 프로젝트에서 공통으로 사용 가능합니다.
